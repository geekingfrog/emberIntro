Introduction to Ember.js

Ember.js is not Ruby on Rails

Ember.js is not a server

MVC in the sense of desktop application

Major concepts in Ember.js

  • Router: Matches the url to an application state.
  • Controller: Handles actions and gives data to the views.
  • Views: Handles users actions (like click).
  • Models: Handle communication with the server (store, retrieve, modify).
  • Templates: Describe html output

Some things to note

  • Ember can generate a lot of code for you (in memory code).
  • Convention over configuration

Different controllers

Ember.Controller: Proxy to its own properties.
A template asking for {{cakeName}} would receive MyController.cakeName


Ember.ObjectController: Proxy to its own properties first and then to its model property. In a template {{name}} would receive MyController.name if it exists and else MyController.model.name.


Ember.ArrayController: Same as an ObjectController, but assumes its model is an array. Provide the each helper and has a length property.
